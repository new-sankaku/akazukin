# バグチェックリスト

## 設定・命名

- [ ] プロパティキー名はコードと設定ファイルで完全一致させる（1件）
- [ ] CSS クラス名のプレフィックス規則を統一する（1件）
- [ ] settings.gradle.kts で同一モジュールを重複 include しない（1件）

## フロントエンド

- [ ] グローバル関数は1箇所だけで定義し、シグネチャを統一する（1件）

## セキュリティ

- [ ] セキュリティヘッダーと CORS 設定は明示的に定義する（1件）
- [ ] パスワードハッシュには十分な強度のアルゴリズムを使用する（1件）
- [ ] 静的リソースのセキュリティ除外パターンは全対象を網羅する（1件）

## エラーハンドリング

- [ ] 例外処理は個別 catch ではなく共通ハンドラに集約する（1件）

## 実装漏れ

- [ ] enum 値を追加したら対応する分岐・処理も必ず実装する（1件）
